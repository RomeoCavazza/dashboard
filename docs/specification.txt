================================================================================
Dashboard Ligue 1 — Technical Specification & Logic Dictionary
================================================================================
Project: Ligue 1 Performance Dashboard
Focus: Football Data Visualization & No-Code Implementation
Host: Vercel (Serverless Functions + Static Hosting)

--------------------------------------------------------------------------------
1. CORE ARCHITECTURE (No-Code vs Code)
--------------------------------------------------------------------------------
Approach:
  Built primarily using Antigravity (No-Code), with custom code for the 
  API Proxy and deployment configuration.

Implementation Layers:
  - UI Library: Custom design system inspired by FootX (dark mode, sleek metrics).
  - Data Sync: Real-time mapping between Antigravity UI components and 
    football-data.org.

--------------------------------------------------------------------------------
2. DATA ENGINE
--------------------------------------------------------------------------------
API Proxy (Vercel Serverless):
  - File: `api/proxy.js`
  - Logic: Acts as a middleware to hide the `X-Auth-Token` from the client side 
    and avoid CORS issues.
  - Endpoints:
    * `/v4/competitions/FL1` (General Info)
    * `/v4/competitions/FL1/standings` (Official Standings)
    * `/v4/competitions/FL1/matches` (Match Results)

Rate Limiting:
  - Free Plan: 10 requests per minute.

--------------------------------------------------------------------------------
3. UI COMPONENTS (Antigravity Blocks)
--------------------------------------------------------------------------------
Visual Logic:
  - Header: Competition meta-data and branding.
  - KPI Cards: Aggregated metrics (Total Matches, Goals, Clean Sheets).
  - Standings Table: Dynamic ranking based on points, goal difference (GD), and 
    W/D/L records.
  - Horizontal Bar Charts: Comparative analysis of Top 5 Teams, Attacks, and 
    Defenses.
  - Histograms: Distribution of goals per matchday.

--------------------------------------------------------------------------------
4. INFRASTRUCTURE
--------------------------------------------------------------------------------
Deployment:
  - Platform: Vercel
  - Root Directory: `04_implementation/dashboard`
  - Environment Variables: `API_KEY` (football-data.org token)

--------------------------------------------------------------------------------
5. VERIFICATION & QUALITY
--------------------------------------------------------------------------------
  - Functional: Real-time data hydration verified against Postman collection.
  - Visual: Vibe coding methodology applied for "FootX" aesthetic alignment.
  - Responsive: Full mobile/tablet support via Antigravity layout engine.
